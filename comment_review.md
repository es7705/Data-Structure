## ch3

### 3-4listS.c

#### [주석-코드 불일치] 51번 줄

```c
// k + 1부터 마지막 원소까지 앞으로 자리 이동
for (i = k; i < n - 1; i++)
```
주석은 "k+1부터"라고 했지만 실제로는 `i = k`부터 시작합니다.
"k번째부터 마지막 전까지 앞으로 한 칸씩 이동"이 맞습니다.

### 3-6ex3_6.c

#### [주석-코드 불일치] 17번, 20번 줄 — 복붙 실수

```c

...
```

## ch5

### 5-3 testPair.c

#### [주석-코드 불일치] 35번 줄 — 번호 누락

```c
// (2)-1 스택에서 마지막으로 읽은 괄호를 꺼냄
open_pair = pop();

// 여는 괄호와 현재 읽고 있는 닫는 괄호의 쌍이 맞는지 확인
if (...)
{
    // (2)-3 괄호 쌍이 틀린 경우 처리
    return 0;
}
else
    break;  // (2)-4로 점프
```
주석에 `(2)-2`가 빠져 있습니다. `(2)-1` 다음 바로 `(2)-3`으로 넘어가서 번호 체계가 맞지 않습니다.

## ch6

### 6-1 queueS.c

#### [주석-코드 불일치] 31번 줄

```c
if (Q->rear == Q_SIZE - 1)   // rear가 최대 크기보다 크면 포화 상태
```
"크면"이 아니라 **"같으면"** 혹은 "마지막 인덱스에 도달하면"이 맞습니다.
`Q_SIZE - 1`은 배열의 마지막 인덱스이고, rear가 거기에 **도달했을 때(==)** 포화입니다.

## Recursion

### 2-14 fact_Recursion.c

#### [주석-코드 불일치] 25~26번 줄

```c
printf("
 fact(1) 함수 호출!");
printf("
 fact(1) 의 값 1 반환!");
```
이 출력은 `n <= 1`인 기저 조건 전체에서 실행됩니다. n이 0이나 음수로 들어와도 "fact(1) 함수 호출!"이라고 출력됩니다.
실제로는 `n <= 1` 전체 분기입니다.
